A.3 Drei direkte Implikationen fuer StockIQ

A.3.2 — Konzentration in High-Conviction-Ideas

Buffett und Abel betonen wiederholt "act quickly and concentrate our capital in a few high conviction ideas". Berkshire haelt >60%% des Aktien-Portfolios in den Top-5-Positionen (Apple, BAC, AmEx, Coke, Chevron — historisch). Das steht im Spannungsverhaeltnis zu einem 269-Ticker-Universum.

Das Universum selbst ist nicht das Portfolio — das war nie die Intention. Aber: Die WL2 (manuelle Watchlist) sollte explizit auf Konzentration optimiert sein.

Konkrete Empfehlung

  1. Konvergenz-Metrik in WL2: Anzeige, wie oft ein Ticker in Top-20 nach verschiedenen Score-Komponenten erscheint (cSc, momSc, fSc, Owner Earnings allein). Wenn ein Ticker in 4 von 4 Top-20-Listen ist → High-Conviction.
  2. Top-N-Konzentrations-KPI: in Synopse anzeigen, wie viel Prozent des hypothetischen 100%-Aktien-Allokations-Equivalents in den Top-10 vs. Top-20 vs. Top-50 entfaellt. Ziel: >40%% in Top-10, gemaess Buffett-Konzentrations-Prinzip.

B — MOVE-Index als zweiter Regime-Filter

B.1 Warum MOVE als Ergaenzung zu VIX?

Der ICE BofA MOVE Index misst implizite Volatilitaet auf US-Treasuries (Optionen auf 2/5/10/30J). Er ist das "VIX-Aequivalent fuer Anleihen" — und reagiert in mehreren historischen Stress-Phasen vor dem VIX.

Schluesselbefund Schwab 2024: "Back in March 2023 when banking industry troubles percolated, the MOVE Index began climbing several days before corresponding moves in the VIX." Genau der Mechanismus, der Apr-Mai 2026 erneut sichtbar war.

B.3 Aktuelle Lage Mai 2026

Per Mai 2026: VIX ~18, MOVE laut letztem Datenpunkt ~70-80 (vom Maerz-Hoch zurueckgekommen). Beide Indikatoren in Normal-Range — kein akuter Stress, aber der Maerz-Spike auf 95 zeigte, dass VIX allein das Bond-Regime nicht abbildet.

Praktischer Befund: Apr-Mai 2026: VIX um 18 (Low), MOVE 80-95 (Elevated). Eine VIX-basierte Regime-Logik haette dieses Risiko-Episode vollstaendig verfehlt. Genau das ist die empirische Begruendung fuer alpha_juno_v7.

C.2 Implementierungs-Skizze (Python, fund_juno v7.9.24)

# fund_juno v7.9.24 — MOVE-Integration
def fetch_move_index():
    """Versuch 1: ^MOVE direkt. Versuch 2: Proxy via ^TNX 5d-Change."""
    try:
        mv = yf.Ticker("^MOVE").history(period="5d")
        if not mv.empty:
            return round(float(mv['Close'].iloc[-1]), 1), str(mv.index[-1].date())
    except Exception:
        pass
    # Proxy: 5d-Change in 10Y-Yield (bps) skaliert
    try:
        tnx = yf.Ticker("^TNX").history(period="10d")
        if len(tnx) >= 6:
            change_bps = (float(tnx['Close'].iloc[-1]) - float(tnx['Close'].iloc[-6])) * 100
            # Empirische Skalierung: 25bps ~ MOVE 100
            move_proxy = 60 + abs(change_bps) * 1.6
            return round(move_proxy, 1), str(tnx.index[-1].date())
    except Exception:
        pass
    return None, None

# In Macro-Block ergaenzen:
macro['move_index'], macro['move_date'] = fetch_move_index()

# alpha_juno v7 — Dual-Regime-Logik (Walk-Forward)
def bond_factor(move):
    if move is None: return 1.0  # neutral wenn keine Daten
    if move < 60:   return 1.00
    if move < 90:   return 0.80
    if move < 120:  return 0.50
    return 0.20

def regime_factor(vix, move):
    return min(vix_factor(vix), bond_factor(move))

# In Signal-Generation:
mom_score_adjusted = mom_score_raw * regime_factor(vix, move)

C.4 Was NICHT mit aufgenommen wird

  • VIX als Entry-Filter: gemaess Whaley 2009 / Banerjee et al. 2007 reagiert VIX asymmetrisch — extreme VIX-Werte koennen contrarian-bullish sein. Daher bleibt VIX nur Momentum-Daempfer, nicht Entry-Filter.
  • Sektor-Faktoren als Regime-Komponente: kein robuster akademischer Konsens. Spaeter, falls 3-Regime-Modell sich als unzureichend erweist.
  • Bonus-Faktor bei sehr ruhigen Maerkten (VIX<15 + MOVE<55): keine Daten-Basis fuer Multiplikator > 1.0. Risiko von Overfit.

D.2 Methodik-Vorschlag

D.2.1 — IC-Mean Weighting (Arxiv 2508.18592)

Das August-2025-Paper "Combined ML for stock selection — dynamic weighting" zeigt: IC-Mean-basierte Gewichtung schlaegt evaluation-metric-basierte Gewichtung in Backtests und Predictive Performance. Empfohlene Formel fuer StockIQ:

W_factor(t) = softmax( IC_mean(factor, t-4Q..t) * ICIR(factor, t-4Q..t) )

# Mit Smoothing (Stabilitaet > schnelle Anpassung):
W_new = 0.7 * W_old + 0.3 * W_target

# Mit Cap (Diversifikations-Schutz):
W_new = clip(W_new, 0.05, 0.50)

D.2.2 — Alpha-Decay-Detection (Arxiv 2509.01393)

Das September-2025-Paper zu adaptivem Alpha-Weighting betont: "trading signals often lose their effectiveness over time due to changes in market conditions [...]. Alpha Decay in quantitative finance."

Konkrete Regel fuer StockIQ:

IF ICIR(factor, rolling 1Y) < 0.5:
    W_factor *= 0.5    # Gewicht halbieren
IF ICIR < 0.0 ueber 2 Quartale:
    W_factor = 0       # Faktor deaktivieren, Alarm im Dashboard

Schutz vor: Faktor-Crowding (zu viele andere Marktteilnehmer nutzen denselben Faktor), Strukturbruch (z.B. ZIRP-Ende), Datenqualitaets-Verfall.

D.2.3 — Korrelations-Check (Pairwise < 0.7)

Gleiches Paper: "We remove alphas with absolute pairwise correlation exceeding 0.7." StockIQ-Faktoren teilweise redundant:

  • Korrelations-Audit: einmalig pruefen, ob EV/EBITDA aus valSc() entfernt werden kann (heute 0.15 Gewicht trotz IC neutral).
