mSig() — Signal-Transformation im Detail ADR-001 v2 • fSc-Filter • HOLD*-Varianten

Pseudocode

mSig(s):                                // s = STOCKS[]-Entry (Ticker-Referenz + Sync-Felder)
  raw = _scoresIdx[s.t].signal          // DIREKT aus Index -- NICHT s.signal (Stale-Risiko!)
  fsc = _scoresIdx[s.t].fund_score      // ADR-001 v2: DIREKT -- NICHT cSc(s)

  // 1. SELL-Pfad: fSc-Filter greift NICHT
  if isSell(raw):                       // sell | sell_zl | sell_ma | sell_hist
    return exit_sig_label(raw)

  // 2. ADR-001 v2: Score-Filter
  if fsc < 70:
    if raw == "hold_dvg":
      return "HOLD DVG"                 // Divergenz hat Vorrang vor fSc-Filter
    return "HOLD*"                      // BUY/STRONG/WATCH --> HOLD* blockiert

  // 3. Voller Score-Pfad
  if raw == "strong":   return "STRONG"
  if raw == "buy":      return "BUY"
  if raw == "hold_dvg": return "HOLD DVG"
  // watch / watch_rsi / buy_rsi_warn / pb: eigene Display-Labels
  return raw

HOLD*-Varianten

raw signal fSc < 70? Ausgabe Bedeutung
hold_dvg egal HOLD DVG Bullish Divergenz — immer HOLD DVG, kein HOLD*
buy / strong / watch / ... ja (<70) HOLD* fSc-Filter aktiv: Score zu niedrig für BUY/STRONG
sell / sell_zl / sell_ma / sell_hist egal SELL-Label SELL-Pfad umgeht fSc-Filter vollständig
